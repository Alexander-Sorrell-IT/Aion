# core-skills

The discipline skills bundled with every install. Process skills (brainstorming, debugging, verification) come before implementation skills (TDD, plans, code review) and shape *how* you approach work, not what you build.

| Skill | Triggered by |
|---|---|
| `brainstorming` | "let's build", "design a system for", before any creative work |
| `test-driven-development` | "add a feature", "fix this bug", before writing implementation code |
| `systematic-debugging` | Any test failure, bug, or unexpected behavior |
| `verification-before-completion` | About to say "done", before committing or opening a PR |
| `writing-plans` | Multi-step task with file-level changes |
| `executing-plans` | Walking through a written plan |
| `code-review` | Reviewing a diff/PR or receiving review feedback |
| `dispatching-parallel-agents` | 2+ independent tasks that can run concurrently |

These skills override default heuristics where they conflict. They're rigid by design — adapting them away from their core discipline usually means abandoning the discipline.
