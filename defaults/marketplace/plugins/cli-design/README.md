# cli-design

Build good CLI tools. Prevents the usual agent-built CLI failure modes: no help text, opaque errors, ambiguous output streams, missing exit codes.

| Skill | Triggered by |
|---|---|
| `cli-design` | Building or reviewing a CLI; "how should this tool be structured?" |

Eight-point checklist: help text, subcommand structure, flag conventions, error messages, output discipline, exit codes, scriptable defaults, common smells.
