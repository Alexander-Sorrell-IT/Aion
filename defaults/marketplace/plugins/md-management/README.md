# md-management

Keep the project memory file (auto-loaded every session) tight and accurate. Anything derivable from `grep`, `git log`, or a README doesn't belong there. Anything not load-bearing for future sessions doesn't belong there. This plugin enforces that discipline.

| Component | Triggered by |
|---|---|
| `md-improver` skill | "update memory", "fix project memory", "audit project doc" |
| `/revise-memory` command | Manual audit + capture pass |
