# observability

Make services observable in production. Two skills covering the two pillars beyond raw logging.

| Skill | Triggered by |
|---|---|
| `structured-logging` | "add logging", "what should I log", "improve logs" |
| `tracing-and-metrics` | "add tracing", "add metrics", "OpenTelemetry", measuring latency/throughput |

Distinguishes signal from noise. Names what to log, what to span, what to measure, what NOT to do. Includes the trap list (high-cardinality labels, sensitive data in logs, average-instead-of-percentile latency).
